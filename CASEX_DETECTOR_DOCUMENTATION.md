# CaseX Detector Implementation Documentation

## Overview

This document explains the `CaseXDetector` class that was added to `sniffin_variation.py`. The detector identifies and reports all `casex` statements in Verilog SystemVerilog code by analyzing the Abstract Syntax Tree (AST) generated by Verible.

## How the Code Works

### 1. AST Parsing

The code uses the `VeribleASTParser` class to parse the text output from Verible's syntax tree generator:

```python
parser = VeribleASTParser()
tree = parser.parse(lines)
```

The parser converts text like:

```
Node @0 (tag: kCaseStatement) {
  Leaf @1 (#"casex" @91-96: "casex")
  ...
}
```

Into a nested dictionary structure:

```python
{
    "type": "kCaseStatement",
    "children": [
        {"type": "casex", "token": "casex", ...},
        ...
    ]
}
```

### 2. Helper Functions

The code provides several utility functions for tree traversal:

- **`find_descendants_by_token(node, token)`**: Finds all leaf nodes with a specific token value
- **`get_children_by_type(node, type_name)`**: Gets direct children of a specific node type
- **`find_descendants(node, type_name)`**: Recursively finds all descendants of a specific type
- **`traverse(node)`**: Generator that yields all nodes in depth-first order
- **`has_child(node, type_name)`**: Checks if a node has a specific child type

### 3. Detector Pattern

All detectors in this script follow the same pattern:

```python
class SomeDetector(SmellDetector):
    def is_target(self, node):
        """Check if this node is what we're looking for"""
        return node["type"] == "kTargetType"

    def extract_info(self, node):
        """Extract relevant information"""
        return ...

    def build_result(self, node):
        """Format the result"""
        return {
            "smell": "detector_name",
            "info": "...",
            "raw": node.get("meta", ""),
        }

    def detect(self, tree):
        """Main detection method"""
        results = []
        for node in traverse(tree):
            if self.is_target(node):
                results.append(self.build_result(node))
        return results
```

## CaseX Detector Implementation

### Class: `CaseXDetector`

The `CaseXDetector` identifies all `casex` statements in the syntax tree.

### Key Methods

#### `is_case_statement(node)`

Checks if a node is a `kCaseStatement` type.

```python
def is_case_statement(self, node):
    return node["type"] == "kCaseStatement"
```

#### `has_casex_keyword(node)`

Determines if the case statement uses `casex` (not `case` or `casez`).

The key insight here is that in the AST, all case statements are represented as `kCaseStatement` nodes, but they contain a leaf token indicating which keyword was used:

- `Leaf (#"case")` for `case`
- `Leaf (#"casez")` for `casez`
- `Leaf (#"casex")` for `casex`

```python
def has_casex_keyword(self, node):
    casex_leaves = find_descendants_by_token(node, "casex")
    return len(casex_leaves) > 0
```

#### `extract_case_expression(node)`

Extracts the expression being tested in the case statement.

For example, in `casex (instruction)`, it extracts the `instruction` expression node.

```python
def extract_case_expression(self, node):
    paren_groups = get_children_by_type(node, "kParenGroup")
    if not paren_groups:
        return None
    expressions = get_children_by_type(paren_groups[0], "kExpression")
    if expressions:
        return expressions[0]
    return None
```

#### `extract_case_expression_token(case_expr)`

Extracts the token/name of the case expression.

For example, extracts `"instruction"` from the expression node.

```python
def extract_case_expression_token(self, case_expr):
    if not case_expr:
        return None
    symbol_ids = find_descendants(case_expr, "SymbolIdentifier")
    if symbol_ids:
        return symbol_ids[0]["token"]
    return None
```

#### `count_case_items(node)`

Counts the number of case items (including default item).

This includes both explicit `kCaseItem` nodes and the optional `kDefaultItem`.

```python
def count_case_items(self, node):
    case_item_lists = get_children_by_type(node, "kCaseItemList")
    count = 0
    for item_list in case_item_lists:
        count += len(get_children_by_type(item_list, "kCaseItem"))
        count += len(get_children_by_type(item_list, "kDefaultItem"))
    return count
```

#### `build_result(case_statement)`

Formats the detection result into a dictionary.

```python
def build_result(self, case_statement):
    case_expr = self.extract_case_expression(case_statement)
    expr_token = self.extract_case_expression_token(case_expr)
    item_count = self.count_case_items(case_statement)

    return {
        "smell": "casex_statement",
        "case_expression": expr_token,
        "num_items": item_count,
        "raw": case_statement.get("meta", ""),
    }
```

#### `detect(tree)`

Main detection method that finds all casex statements in the tree.

```python
def detect(self, tree):
    results = []
    for node in traverse(tree):
        if not self.is_case_statement(node):
            continue
        if not self.has_casex_keyword(node):
            continue
        results.append(self.build_result(node))
    return results
```

## AST Structure Example

### Verilog Code

```verilog
module testbench;
    logic [3:0] instruction;
    logic [2:0] opcode;

    always_comb begin
        casex (instruction)
            4'b0???:
                opcode = 3'b001;
            4'b1000:
                opcode = 3'b001;
            default:
                opcode = 3'b001;
        endcase
    end
endmodule
```

### Corresponding AST Structure (simplified)

```
Node (tag: kCaseStatement)
├── Leaf (#"casex")                    <- keyword leaf
├── Node (tag: kParenGroup)            <- contains expression
│   └── Node (tag: kExpression)        <- the tested expression
│       └── Node (tag: kFunctionCall)
│           └── Leaf (#SymbolIdentifier: "instruction")
├── Node (tag: kCaseItemList)          <- contains case items
│   ├── Node (tag: kCaseItem)          <- first case item
│   ├── Node (tag: kCaseItem)          <- second case item
│   └── Node (tag: kDefaultItem)       <- default item
```

## Usage Example

### Using the detector directly:

```python
from sniffin_variation import *

# Read the syntax tree
with open('atom.syntax_tree.txt', 'r', encoding='utf-8') as f:
    lines = f.readlines()

# Parse
parser = VeribleASTParser()
tree = parser.parse(lines)

# Create detector
detector = CaseXDetector()

# Detect casex statements
results = detector.detect(tree)

# Print results
for result in results:
    print(f"Found casex: {result['case_expression']}")
    print(f"  Number of case items: {result['num_items']}")
```

### Using with the Analyzer:

```python
detectors = [
    UnsizedBaselessLiteralDetector(),
    NonAutomaticFunctionDetector(),
    PackedMultidimensionalConcatAssignmentDetector(),
    CaseXDetector(),  # New detector
]

analyzer = Analyzer(detectors)
results = analyzer.run(tree)

# Results include casex_statement entries
for result in results:
    if result['smell'] == 'casex_statement':
        print(f"CaseX: {result['case_expression']}")
```

## Output Format

The detector returns a list of dictionaries with the following structure:

```python
{
    "smell": "casex_statement",
    "case_expression": "instruction",  # The variable being tested
    "num_items": 3,                     # Total number of case items (including default)
    "raw": "Node @0 (tag: kCaseStatement) { ... }",  # Raw AST node
}
```

### Example Output

```
Found casex statement:
  Case Expression: instruction
  Number of case items: 3
```

## Key Design Features

1. **Differentiation from case/casez**: Uses token matching to distinguish between case types
2. **Expression extraction**: Captures the variable being tested (e.g., "instruction")
3. **Item counting**: Counts all case alternatives including the default case
4. **Extensibility**: Follows the same pattern as other detectors for easy maintenance
5. **Consistent interface**: Returns results in the same format as other detectors

## How It Differs from Other Detectors

| Detector                                         | Purpose                                     | Target                            | Output                         |
| ------------------------------------------------ | ------------------------------------------- | --------------------------------- | ------------------------------ |
| `UnsizedBaselessLiteralDetector`                 | Finds numeric literals without size/base    | Numbers in control structures     | literal, context               |
| `NonAutomaticFunctionDetector`                   | Finds functions without `automatic` keyword | Functions                         | function name                  |
| `PackedMultidimensionalConcatAssignmentDetector` | Finds problematic assignments               | Multidimensional packed variables | variable name                  |
| `CaseXDetector`                                  | **Finds all casex statements**              | **casex keywords**                | **case_expression, num_items** |
